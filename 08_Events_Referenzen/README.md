# 📁 Events-Referenzen — Sortierung & Workflow

Sammelt alle Foto-/Video-Materialien aus echten Events. **Pro Event ein eigener Unterordner** mit dreistufiger Struktur — damit du jederzeit weißt was wo liegt und was wo benutzt wurde.

---

## 📂 Ordner-Struktur pro Event

```
2026-MM_KurzName_AnlassTyp/
├── 01_Roh/              ← Unbearbeitete Originale (vom Phone, Kamera)
├── 02_Bearbeitet/       ← Zugeschnittene / optimierte Versionen
├── 03_Verwendet/        ← Versionen, die auf Webseite / Insta etc. live sind
└── README.md            ← Event-Steckbrief (Template siehe unten)
```

**Naming-Konvention:** `YYYY-MM_KurzName_AnlassTyp`

Beispiele:
- `2026-04_Möbel-Ehrmann_Tag-der-offenen-Tür`
- `2026-04_Deutsche-Bank_Mitarbeiterveranstaltung`
- `2026-06_Schmidt-Müller_Hochzeit`
- `2026-08_Saarbrücker-Sommerfest_Stadtfest`

---

## 📝 README-Template pro Event

Lege in jedem Event-Ordner eine `README.md` mit folgendem Inhalt an (Copy & Paste, dann ausfüllen):

```markdown
# [Event-Name] — [Datum]

## Eckdaten
- **Datum:** YYYY-MM-DD
- **Veranstalter:** [Firma / Brautpaar / Verein]
- **Anlass:** [Hochzeit / Firmenfeier / Tag der offenen Tür / Messe / Stadtfest]
- **Ort:** [Stadt / Location]
- **Gäste:** ca. XYZ
- **Eventzeit:** X Stunden
- **Sofortprints:** XYZ Stück
- **Paket:** Eventabend / Tagesevent

## Highlights
- [Was war besonders an diesem Event?]
- [Anekdoten, Wow-Momente, ungewöhnliche Wünsche]

## Quote / Bewertung
- [Falls Google-/Bewertung erhalten: hier kopieren]
- Bewerter: [Name + Position]

## Verwendet auf
- [ ] Webseite: [welche Sektion/Datei]
- [ ] Insta: [Post-Datum / Story-ID]
- [ ] Google Business: [Beitrag]
- [ ] Marketing-Material: [Flyer, Präsentation]

## Notizen
- [interne Notizen, Kontaktdaten Veranstalter, Sonderwünsche]
```

---

## 🚦 Workflow

1. **Nach jedem Event** → Neuen Unterordner anlegen mit dem Naming-Schema
2. **Roh-Material vom Phone/Kamera** → `01_Roh/` (alles, keine Selektion)
3. **Best-of bearbeiten** → `02_Bearbeitet/` (Crops, Color, Anonymisierung falls nötig)
4. **Live-Versionen markieren** → `03_Verwendet/` (Kopie aus 02_, sobald veröffentlicht)
5. **README.md ausfüllen** — auch nur halb, ist besser als gar nicht

---

## 🎯 Vorteile dieser Struktur

| Brauchst du... | Findest du in... |
|---|---|
| Original vom Phone | `01_Roh/` |
| Bearbeitete Variante für Insta | `02_Bearbeitet/` |
| Was schon live ist | `03_Verwendet/` |
| Eckdaten des Events | `README.md` |
| Quotes für Werbung | `README.md` → Quote-Sektion |
